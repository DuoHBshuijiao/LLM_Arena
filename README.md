# LLM Arena

**同一道题，多路模型，一次看清差异。**

在本地浏览器里完成：出题 → 并行生成 → 多评委打分 → 人工校准 → 图表对比与成绩存档。密钥与上游地址由你自备，界面只负责把流程跑顺、把结果留痕。

---

## 你能用它做什么

- **并排评测**：为多个模型配置同一套任务与采样参数，流式输出同屏对照，省去多窗口来回切换。
- **评委与汇总**：支持多轮评委打分、可选汇总模型，并混入人工分得到最终排序。
- **结果可沉淀**：一键保存快照到本机 `data/scores/`，历史记录可回看、可导出 JSON，图表页支持柱状对比。
- **兼容常见 API**：通过本地代理转发 OpenAI 风格的 Chat Completions 与模型列表，降低浏览器直连与跨域带来的麻烦。

---

## 快速开始

**环境**：已安装 [Node.js](https://nodejs.org/)（建议当前 LTS）。

```bash
git clone https://github.com/DuoHBshuijiao/LLM_Arena.git
cd LLM_Arena
npm install
npm run dev
```

浏览器访问 **http://localhost:9300/**。前端开发服务默认 **9300**，本地代理默认 **9400**；开发时前端会将 `/api` 请求转发到代理服务。

**Windows**：双击根目录 `start.bat` 可安装依赖、启动开发进程并尝试打开上述地址。

单独启动：

```bash
npm run dev:server   # 仅代理与成绩 API
npm run dev:client   # 仅前端（需代理已就绪，否则健康检查会提示未连接）
```

生产构建目前包含前端静态资源；部署时请将 **`/api` 反向代理**到同一套代理服务（或等价路由），与开发环境下 Vite 的行为一致。

---

## 使用之前

1. 在 **设置** 中填写各 API 的 Base URL、密钥与模型；支持拉取模型列表后勾选写入。
2. 评测与打分均在浏览器会话与本地服务间完成；**请妥善保管密钥**，勿将含密钥的截图或导出文件公开分享。
3. 连接状态在顶栏显示；若显示未连接，请确认代理进程已启动或检查端口占用。

---

## 数据放在哪

- **浏览器**：配置与运行状态通过本地存储持久化（键名 `llm-arena`），清除站点数据会一并清空。
- **本机磁盘**：通过「保存成绩」写入仓库下 `data/scores/*.json`，便于备份与版本管理时自行忽略或纳入。
