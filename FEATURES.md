# LLM Arena 功能清单与演进方向

本文档梳理当前版本已实现能力，并列出**高概率**后续需求（便于排期与架构预留）。

---

## 一、当前功能清单

### 1. 配置与连接

- 多 **API 预设**（Base URL、Key、并发上限、手动/拉取的模型 ID 列表）。
- 通过本地代理 **拉取模型列表**（多路径兼容）并写入预设。
- 顶栏 **健康检查**（`/api/health`）与手动重试。
- 全局采样参数：temperature、max tokens、top-p。

### 2. 任务与评测预设

- 内置多类 **评测题目预设**（切换时套用题目与评委模板等）。
- **自定义题目**条目（持久化在本地存储）。
- 任务提示词与预设联动（自定义题不覆盖汇总文案等策略见界面说明）。

### 3. 运行与流水线

- 多模型、多样本 **并行生成**（按预设并发限制）。
- 流式输出同屏对照；**生成 → 多评委 → 可选汇总** 树状画布。
- 画布 **平移、缩放、键盘操作**；导出 **PNG**。
- 单线程：**暂停 / 恢复 / 取消 / 单槽位中止 / 弃用线程**。
- **批量重试失败线程**（对仍存在 `failedPipelineStep` 的失败线程顺序恢复）。
- 整轮 **取消评测**（确认对话框）。
- **清空结果**、预估 API 调用次数、就绪状态校验。

### 4. 打分与汇总

- 每线程填写 **评委分**、**线程人工分**。
- **分数计算器**：评委权重、人工分权重、按模型最终均分（与 `scoreCalculations` 一致）。
- **图表页**：按模型整体验收人工分、与自动汇总柱状对比；排序（模型 / 自动 / 人工）。
- **批量清空**整体验收人工分（确认对话框，不影响线程内分数）。

### 5. 持久化与导出

- 配置与运行相关状态 **localStorage**（`llm-arena`，含版本迁移）。
- 评测完成后 **保存成绩快照** 至仓库 `data/scores/*.json`（经本地服务 POST）。
- **历史成绩**列表加载、单条下载 JSON、**打包下载全部**。
- 图表与历史只读查看。

### 6. 无障碍与操作效率

- **跳过链接**至主内容。
- 主导航 `aria-current`、表格 `caption`、错误 `role="alert"`、加载 `aria-busy` 等既有标记。
- **确认框**：焦点落入对话框、Tab 循环、关闭后还原焦点、`role="dialog"`。
- **键盘快捷键**：`Alt+Shift+1/2/3` 切换标签；`F1` 或 `Alt+Shift+/` 展开快捷键说明（输入框内不拦截）。

### 7. 服务端（Node 代理）

- OpenAI 兼容 **Chat Completions 流式代理**（多路径 404 回退）。
- **模型列表** GET 多路径尝试。
- 成绩目录 **读写**；`GET /scores` 并行读取 JSON 文件后按 `savedAt` 排序。

## 二、推测的高概率新需求

| 方向 | 说明 |
|------|------|
| 鉴权与多用户 | 本地工具若上公网，需登录、令牌与数据隔离。 |
| 评测队列 / 批次 | 多道题按队列跑、批量导出对比表。 |
| 自定义评委维度 | 非单一分数，而是多子维度 rubric 与加权。 |
| 成本统计 | 按模型估算 token 费用、调用次数报表。 |
| 导入题目 / 从剪贴板 | 快速载入 JSON/CSV 任务集。 |
| 深色主题与对比度 | 系统偏好 `prefers-color-scheme` 与 WCAG 对比度调优。 |
| 国际化 | 中英 UI 切换、日期与数字格式。 |
| 离线/纯前端模式 | 无 Node 时仅浏览器 + 直连 API（CORS 自理）降级提示。 |

当前迭代已覆盖其中与「效率、批量、无障碍」重叠的部分（快捷键、批量重试、批量清空人工分、对话框焦点管理、服务端列表读取并行化）。
